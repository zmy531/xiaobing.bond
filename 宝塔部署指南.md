# 宝塔面板部署超详细指南

---

## 📋 准备工作

### 你需要准备的东西：

1. ✅ 一台服务器（推荐腾讯云/阿里云/华为云，1核1G足够）
2. ✅ 一个域名（比如 `xiaobing.bond`）
3. ✅ 宝塔面板已经安装在服务器上

### 如果还没装宝塔面板：

**安装命令（复制到服务器终端执行）：**

CentOS系统：
```bash
yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh
```

Ubuntu/Debian系统：
```bash
wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh
```

安装完成后会显示面板地址、用户名、密码，保存好。

## 🌐 一、域名解析

在部署之前，先把域名解析到你的服务器IP。

1. 登录你的域名注册商控制台（比如阿里云、腾讯云、Namecheap等）
2. 找到「域名解析」或「DNS管理」
3. 添加两条解析记录：

| 主机记录 | 记录类型 | 记录值 |
|----------|----------|--------|
| @ | A | 你的服务器IP |
| www | A | 你的服务器IP |

> 解析生效一般需要几分钟到几小时，可以用 `ping 你的域名` 测试是否生效

---

## 🖥️ 二、登录宝塔面板

1. 在浏览器打开宝塔面板地址（一般是 `http://服务器IP:8888`）
2. 输入用户名和密码登录
3. 首次登录会让你安装套件，选择：
   - **Nginx**（推荐）或 Apache**
   - **PHP 7.4 或更高版本
   - MySQL不用选（本项目不需要数据库）
   - 点击「一键安装」

> 等待安装完成，可能需要5-10分钟

---

## 📂 三、创建网站

### 第1步：添加站点

1. 点击左侧菜单「网站」
2. 点击「添加站点」
3. 填写信息：

```
域名：xiaobing.bond （你的域名）
根目录：/www/wwwroot/xiaobing.bond （自动生成，不用改）
FTP：不创建
数据库：不创建（选MySQL，因为本项目不用数据库
PHP版本：选择 PHP-74 或 PHP-81 （选7.4或更高都可以）
```

4. 点击「提交」

### 第2步：确认网站创建成功

在网站列表里能看到你刚创建的网站，状态是「运行中」。

## 📤 四、上传文件

### 方式一：宝塔文件管理器上传（适合文件少的情况

1. 点击网站右侧的「根目录」按钮，进入文件管理器
2. 全选里面默认的文件（index.html、404.html等），删除
3. 点击「上传」按钮
4. 把你本地 `个人主页` 文件夹里的**所有文件**都选中上传

> ⚠️ 注意：是上传 `个人主页` 文件夹**里面**的文件，不是上传整个文件夹！

上传完成后，根目录里应该是这样的：
```
/www/wwwroot/xiaobing.bond/
├── index.html
├── login.html
├── admin.html
├── message.html
├── api/          ← 这个文件夹必须有
├── static/
├── data/
├── docs/
├── tools/
├── server/
├── 背景乐.mp3
├── Caddyfile
├── Dockerfile
├── docker-compose.yaml
└── ...
```

### 方式二：FTP上传（文件多时更快

1. 宝塔面板 → 软件商店 → 安装「Pure-Ftpd」
2. 网站 → FTP → 创建FTP账号
3. 用 FileZilla 等FTP软件连接上传

## 🔧 五、设置目录权限

这一步很重要！不然后台数据保存不了、图片上传不了。

### 需要写权限的目录：

- `data/` - 存放数据JSON文件
- `uploads/` - 存放上传的图片、语音等（如果没有就新建）

### 设置方法一：宝塔文件管理器（推荐，简单）

1. 进入网站根目录
2. 找到 `data` 文件夹
3. 右键 → 权限
4. 权限设置为 `755`
5. 勾选「应用到子目录」
6. 所有者改成 `www`（如果有这个选项）
7. 点击「确认」

8. 同样的方法设置 `uploads` 文件夹（如果没有，先新建一个）

### 设置方法二：SSH终端命令

1. 宝塔面板 → 终端
2. 输入以下命令：

```bash
# 进入网站目录
cd /www/wwwroot/xiaobing.bond

# 创建uploads目录（如果没有的话）
mkdir -p uploads

# 设置data目录权限
chmod -R 755 data
chown -R www:www data

# 设置uploads目录权限
chmod -R 755 uploads
chown -R www:www uploads
```

## ✅ 六、测试是否正常

### 测试1：首页能不能打开

浏览器访问：`http://你的域名`

- ✅ 能看到主页 → 正常
- ❌ 404 Not Found → 文件没传错位置了
- ❌ 500错误 → 看下面常见问题

### 测试2：API能不能用

浏览器访问：`http://你的域名/api/health.php`

应该返回类似这样的JSON：
```json
{"status":"ok","time":1234567890}
```

- ✅ 返回上面的内容 → PHP正常工作
- ❌ 404 → api文件夹没上传对
- ❌ 显示PHP代码 → PHP没启用，检查PHP版本
- ❌ 500错误 → 看错误日志

### 测试3：后台能不能登录

浏览器访问：`http://你的域名/login.html`

输入默认账号密码：
- 用户名：`zmy`
- 密码：`080531`

- ✅ 登录成功，进入管理后台 → 一切正常
- ❌ 提示「后端服务异常」→ API路径不对
- ❌ 提示「用户名或密码错误」→ 检查账号密码

## 🔐 七、修改管理员密码（必做！）

默认密码大家都知道，一定要改！

1. 宝塔文件管理器进入网站根目录
2. 打开 `api/login.php` 文件
3. 找到第6-8行左右：

```php
define('ADMIN_USER', 'zmy');                    // 用户名
define('ADMIN_PASS', '080531');              // 密码
define('JWT_SECRET', 'zmy_admin_secret_key_2026');  // 密钥，随便改
```

4. 修改成你自己的：
```php
define('ADMIN_USER', '你的用户名');
define('ADMIN_PASS', '你的密码');
define('JWT_SECRET', '随便打一串乱码就行');
```

5. 保存文件，立即生效，不用重启。

## 🔒 八、配置HTTPS（强烈建议）

没有HTTPS的网站浏览器会提示「不安全」，而且很多功能也用不了。

### 申请免费SSL证书：

1. 宝塔面板 → 网站 → 你的网站 → 点击「SSL」
2. 选择「Let's Encrypt」
3. 勾选你的域名（@和www都勾上）
4. 勾选「自动续期」
5. 点击「申请」

等待几十秒，申请成功后：

1. 右上角「强制HTTPS」打开 ✅

这样访问 `http://你的域名` 会自动跳转到 `https://你的域名`

## 🆘 十、常见问题排查

### 问题1：访问网站显示 404 Not Found

**可能原因：**
- 文件没上传到正确位置
- 网站根目录设置错了

**解决方法：
1. 检查网站设置里的根目录是不是 `/www/wwwroot/你的域名`
2. 进入根目录，看看有没有 index.html 文件
3. 确认 api 文件夹在根目录下

---

### 问题2：访问 /api/health.php 显示 404

**可能原因：**
- api 文件夹没上传
- 文件名大小写不对（Linux区分大小写）

**解决方法：**
1. 确认根目录下有 `api` 文件夹
2. 确认 `api` 文件夹里有 `health.php` 文件
3. 注意文件名大小写：`health.php` 不是 `Health.php`

---

### 问题3：访问PHP文件直接下载文件而不是执行

**可能原因：**
- PHP没安装
- PHP版本没选对

**解决方法：**
1. 宝塔 → 网站 → 设置 → PHP版本
2. 选择 PHP-74 或更高
3. 保存

---

### 问题4：登录后台提示「后端服务异常」

**可能原因：**
- API路径不对
- PHP文件有语法错误

**排查方法：**
1. 直接访问 `/api/login.php` 看看返回什么
2. 如果是500错误，看PHP错误日志
3. 宝塔 → 软件商店 → PHP设置 → 错误日志

---

### 问题5：留言板图片上传失败

**可能原因：**
- uploads 目录不存在
- uploads 目录没有写权限
- PHP上传大小限制太小

**解决方法：**

```bash
cd /www/wwwroot/你的域名
mkdir -p uploads
chmod 755 uploads
chown www:www uploads
```

如果还是不行，修改PHP上传限制：
1. 宝塔 → 软件商店 → PHP设置 → 上传限制 → 改成 50M 或更大

---

### 问题6：后台修改数据不保存

**可能原因：**
- data 目录没有写权限

**解决方法：**
```bash
cd /www/wwwroot/你的域名
chmod -R 755 data
chown -R www:www data
```

---

### 问题7：访客IP显示都是127.0.0.1或者都是北京

**可能原因：**
- 反向代理没有正确转发真实IP
- IP定位接口不准

**解决方法：**
1. 如果用的是Cloudflare等CDN，检查CDN设置里确保开启「真实IP」
2. 本项目已经配置了4个IP定位接口，会自动降级

---

### 问题8：音乐播放没有声音

**正常现象！**
现在浏览器都禁止自动播放有声内容，必须用户点击一次页面后才会播放。

用户访问网站后，随便点一下页面任意位置，音乐就会响了。

---

### 问题9：连续刷新页面，访客记录不增加

**正常设计！**
同一IP访问同一个页面，2分钟内只记录一次，防止刷量。

访问不同的页面会分别记录。

---

### 问题10：怎么修改主页上的文字？

直接编辑 `index.html` 文件，找到对应的文字修改就行。

- 头像：替换 `static/img/logo.png`
- 名字：找 `index.html` 里的名字
- 标签：找 `left-tag-item` 相关的
- 时间线：找 `left-time` 里的内容

## 📱 十一、备份数据

定期备份很重要！

### 备份方法：

1. 宝塔 → 网站 → 你的网站 → 备份
2. 点击「备份网站」
3. 定期下载备份文件到本地

### 需要备份的文件：
- `data/` 目录 - 所有数据
- `uploads/` 目录 - 上传的图片语音
- `static/img/logo.png` - 你的头像logo
- `index.html` - 你修改过的主页

---

## 🎯 十二、更新版本

以后有新版本时怎么更新：

1. 先备份！先备份！先备份！
2. 上传新文件覆盖旧文件
3. 注意不要覆盖 `data/` 和 `uploads/` 目录
4. 不要覆盖你修改过的 `index.html`

## 💡 小技巧

### 怎么快速查看错误：
1. 浏览器按 F12 打开开发者工具
2. 点「Console」看有没有报错
3. 点「Network」看请求失败的请求

有问题先看错误信息，大部分问题都能自己解决！

## 🚀 部署教程

### 方式一：宝塔面板部署（推荐，最简单）

> 适合有宝塔面板的虚拟主机/服务器用户，上传即用，无需配置。

#### 前置要求

- 宝塔面板（或其他面板）
- PHP 7.4 及以上版本
- 支持 cURL 扩展（IP定位需要）

#### 详细步骤

##### 第1步：创建网站

1. 登录宝塔面板
2. 点击左侧「网站」→「添加站点」
3. 填写信息：
   - **域名**：你的域名，例如 `xiaobing.bond`
   - **根目录**：`/www/wwwroot/xiaobing.bond`
   - **FTP**：不需要
   - **数据库**：不需要（本项目用JSON文件存储）
   - **PHP版本**：选择 PHP-74 或更高
4. 点击「提交」

##### 第2步：上传文件

1. 点击网站根目录进入文件管理器
2. 把 `个人主页` 文件夹里的**所有文件**上传到根目录
3. 上传完成后目录结构应该是：
   ```
   /www/wwwroot/xiaobing.bond/
   ├── index.html
   ├── login.html
   ├── admin.html
   ├── api/
   ├── static/
   ├── data/
   └── ...
   ```

##### 第3步：设置目录权限

以下目录需要**写入权限**：

```bash
# 进入网站目录
cd /www/wwwroot/xiaobing.bond

# 创建必要目录
mkdir -p data uploads

# 设置权限
chmod -R 755 data uploads
chown -R www:www data uploads
```

或者在宝塔文件管理器中：
- 右键 `data` 文件夹 → 权限 → 755 → 应用到子目录
- 右键 `uploads` 文件夹 → 权限 → 755 → 应用到子目录

##### 第4步：测试是否正常

1. **测试API**：访问 `https://你的域名/api/health.php`
   - 应该返回类似 `{"status":"ok","time":1234567890}`
   - 如果返回404，说明文件没上传对

2. **测试首页**：访问 `https://你的域名/`
   - 应该能看到主页

3. **测试后台登录**：访问 `https://你的域名/login.html`
   - 默认账号：`zmy`
   - 默认密码：`080531`

##### 第5步：修改管理员密码（重要！）

编辑 `api/login.php` 文件，修改以下内容：

```php
// 第6-8行左右
define('ADMIN_USER', 'zmy');                    // 改成你的用户名
define('ADMIN_PASS', '080531');                  // 改成你的密码
define('JWT_SECRET', 'zmy_admin_secret_key_2026');  // 改成随机字符串（随便改）
```

修改完保存即可生效，不需要重启。

##### 第6步：配置HTTPS（强烈建议）

1. 宝塔面板 → 网站 → 你的网站 → SSL
2. 选择「Let's Encrypt」→ 勾选域名 → 申请
3. 申请成功后，右上角开启「强制HTTPS」

##### 第7步：配置伪静态？不需要！

本项目所有API路径都带 `.php` 后缀，**不需要配置伪静态**，上传即用。

## ❓ 常见问题

### Q1：访问 /api/ 报 404

**原因：** PHP文件没上传，或路径不对

**解决：**
- 确认 `api/login.php` 等文件已上传
- 确认文件在网站根目录的 `api` 文件夹下
- 确认PHP版本 ≥ 7.4

### Q2：登录提示「后端服务异常」

**原因：** API文件路径不对，或PHP不工作

**解决：**
1. 直接访问 `/api/health.php` 看是否返回JSON
2. 如果返回404，检查文件是否上传
3. 如果显示PHP代码，说明PHP没启用

### Q3：留言板图片上传失败

**原因：** uploads 目录没有写入权限

**解决：**
```bash
mkdir -p /www/wwwroot/你的域名/uploads
chmod 755 /www/wwwroot/你的域名/uploads
chown www:www /www/wwwroot/你的域名/uploads
```

### Q4：访客IP显示北京/不准确

**原因：**
- 没有正确获取真实客户端IP（反向代理场景）
- IP定位接口返回不准确

**解决：**
- 检查Caddy/Nginx是否配置了 `X-Forwarded-For` 头
- 本项目已配置4个IP定位接口自动降级，一般能正确显示

### Q5：音乐播放没声音

**原因：** 浏览器自动播放策略限制，需要用户首次交互后才能播放

**解决：** 正常现象，用户点击页面任意位置后会自动播放

### Q6：后台数据不保存/修改无效

**原因：** `data` 目录没有写入权限

**解决：**
```bash
chmod -R 755 /www/wwwroot/你的域名/data
chown -R www:www /www/wwwroot/你的域名/data
```

### Q7：连续访问同一个页面，访客记录不增加

**正常设计：** 同一IP访问同一页面，2分钟内只记录一次，避免刷量。
访问不同页面会分别记录。

### Q8：怎么修改主页内容？

- **文字内容**：直接编辑 `index.html`
- **主题颜色**：编辑 `static/css/root.css` 里的CSS变量
- **头像/logo**：替换 `static/img/logo.png`
- **标签、时间线**：编辑 `index.html` 对应部分

## 🔧 技术栈

| 分类 | 技术 |
|------|------|
| 前端 | HTML5、CSS3、原生JavaScript |
| 后端 | PHP 7.4+ |

**为什么用JSON文件存储而不用数据库？**
- 个人站点数据量小，JSON完全够用
